1. Introduction & Syntax

This section introduces the fundamental role of JavaScript in web development. You'll learn what JavaScript is, how to add it to a webpage, and the basic syntax that forms the building blocks of any script. Understanding these core ideas is the first step to making your websites interactive.

What is JavaScript?

Think of a website as a house. HTML is the frame and structure of the house. CSS is the paint, furniture, and decorations that make it look good. JavaScript is the electricity, plumbing, and appliances that make the house functionalâ€”it lets you turn on lights, open doors, and interact with it. It's a programming language that brings websites to life.

How to Add JavaScript to a Webpage

Internal Script: You can write JavaScript directly inside your HTML file within <script> tags.

<script>
  console.log("This code is inside the HTML file.");
</script>

External Script: For larger projects, it's best to keep your JavaScript in a separate .js file and link it. This is the most common and recommended method.

<!-- In your index.html file -->
<script src="myscript.js"></script>

Statements & Comments

A JavaScript program is a list of instructions, called statements. It's a good practice to end each statement with a semicolon (;).

Comments are notes in your code that are ignored by the browser. They are for humans to read.

// This is a single-line comment.

/*
  This is a
  multi-line
  comment.
*/

2. Variables & Data Types

Variables are the fundamental storage units in programming. This section explains how to declare variables using `let`, `const`, and `var`, and introduces the core data types that determine what kind of information you can store, from simple text and numbers to complex objects and lists.

Declaration Keywords

  • let: Use this when you expect the value to change. It is block-scoped.
  • const: Use this for values that will never change (constants). It is also block-scoped.
  • var: The old way. It is function-scoped, which can lead to unexpected behavior. It's best to use let and const.

Data Types

Primitive Types

  • String: Text, wrapped in quotes. let greeting = "Hello";
  • Number: Any number, including integers and decimals. let age = 30;
  • Boolean: Represents true or false. let isComplete = true;
  • Undefined: A variable that has been declared but not assigned a value. let user;
  • Null: Represents the intentional absence of a value. let selectedUser = null;

Object Type

Object: A collection of key: value pairs.

let person = {
  firstName: "Bob",
  age: 42
};
console.log(person.firstName); // Dot notation
console.log(person["age"]);    // Bracket notation

Array: A numbered list of items.

let colors = ["red", "green", "blue"];
console.log(colors[0]); // Access the first item: "red"
colors.push("yellow");  // Add an item to the end

3. Operators

Operators are the symbols that perform actions in JavaScript, from basic math to complex logical comparisons. This section covers the essential operators you'll use to manipulate data, make decisions, and write functional code.

  • Arithmetic: +, -, *, /, % (remainder), ** (exponentiation).
  • Assignment: =, +=, -=, *=, /=.
  • Comparison: === (strict equality), !== (strict inequality), >, <, >=, <=. Always prefer strict equality (===) over loose equality (==).
  • Logical: && (AND), || (OR), ! (NOT).
  • Ternary Operator: A shortcut for an if statement. let result = (condition) ? 'value if true' : 'value if false';

4. Control Flow

Control flow structures allow you to make decisions and direct the execution path of your code. Here, you'll learn about `if/else` statements for conditional logic and the `switch` statement for handling multiple distinct cases, which are essential for creating dynamic and responsive applications.

if...else if...else

let hour = 14;
if (hour < 12) {
  console.log("Good morning");
} else if (hour < 18) {
  console.log("Good afternoon");
} else {
  console.log("Good evening");
}

switch Statement

let day = "Tuesday";
switch (day) {
  case "Monday":
    console.log("Start of the work week.");
    break;
  case "Friday":
    console.log("End of the work week!");
    break;
  default:
    console.log("It's a regular day.");
}

5. Loops

Loops are a fundamental concept for performing repetitive tasks efficiently. This section explores the different types of loops in JavaScript, including the `for` loop for a known number of iterations, the `while` loop for conditional repetition, and the modern `for...of` and `for...in` loops for iterating over arrays and objects.

for loop

for (let i = 0; i < 5; i++) {
  console.log("Iteration " + i);
}

for...of loop

const fruits = ["Apple", "Banana", "Cherry"];
for (const fruit of fruits) {
  console.log(fruit);
}

for...in loop

const user = { name: "Alice", age: 30 };
for (const key in user) {
  console.log(`${key}: ${user[key]}`);
}

6. Functions

Functions are reusable blocks of code that form the core of any JavaScript application. This section covers how to define and call functions using different syntaxes (declarations, expressions, and arrow functions), helping you write modular, organized, and maintainable code.

Function Declaration

function add(num1, num2) {
  return num1 + num2;
}

Function Expression

const subtract = function(num1, num2) {
  return num1 - num2;
};

Arrow Function (ES6)

const multiply = (num1, num2) => num1 * num2;

7. Scope

Scope determines where variables can be accessed in your code. Understanding global, function, and block scope is crucial for preventing bugs and managing your application's state. This section clarifies these concepts and explains why modern `let` and `const` are preferred over `var`.

let globalVar = "I'm global";

function myFunction() {
  let functionVar = "I'm in a function";
  if (true) {
    let blockVar = "I'm in a block";
    console.log(blockVar);    // Accessible
    console.log(functionVar); // Accessible
  }
  // console.log(blockVar); // ERROR! Not accessible here.
}

8. The 'this' Keyword

The `this` keyword is one of JavaScript's most powerful and sometimes confusing features. Its value is determined by how a function is called (its execution context). This section demystifies `this` by explaining its behavior in object methods, regular functions, and arrow functions, giving you the clarity needed to use it effectively.

In an Object Method

this refers to the object the method is called on.

const person = {
  name: "Carlos",
  sayName() { console.log(this.name); }
};
person.sayName(); // 'this' is 'person'. Output: Carlos

In an Arrow Function

Arrow functions don't have their own this. They inherit it from their parent scope. This is extremely useful in many situations, like event listeners.

9. Prototypes and Prototypal Inheritance

JavaScript's inheritance model is based on prototypes. Every object has a link to a prototype object, forming a "prototype chain." This section explains how this powerful mechanism works, allowing objects to inherit properties and methods from other objects.

const animal = {
  eats: true,
  walk() { console.log("Animal is walking."); }
};
const rabbit = Object.create(animal);
rabbit.jumps = true;

console.log(rabbit.eats); // true (from prototype)
rabbit.walk();            // "Animal is walking." (from prototype)

10. Classes (ES6)

Classes are a modern blueprint for creating objects, providing a cleaner syntax over prototype-based inheritance. This section shows how to define classes, use constructors for initialization, and implement inheritance with `extends` and `super`.

class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} makes a noise.`);
  }
}
class Dog extends Animal {
  constructor(name, breed) {
    super(name); // calls the parent class's constructor
    this.breed = breed;
  }
  speak() {
    console.log(`${this.name} barks.`);
  }
}
const myDog = new Dog('Rex', 'German Shepherd');
myDog.speak(); // Rex barks.

11. Asynchronous JavaScript

JavaScript is single-threaded, but it can handle long-running tasks without freezing the user interface using asynchronous operations. This section explains the modern `async/await` syntax, which makes handling asynchronous code like fetching data from a server clean and readable.

async function fetchUserData() {
  try {
    const response = await fetch('https://api.github.com/users/github');
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    const data = await response.json();
    console.log(data.name); // GitHub
  } catch (error) {
    console.error("Could not fetch data:", error);
  }
}
fetchUserData();

12. Closures

A closure is when a function "remembers" the variables from the scope where it was created, even if it's executed in a different scope. This powerful feature is essential for creating private variables and stateful functions, as demonstrated in the counter example.

function createCounter() {
  let count = 0; // This variable is "private"

  return {
    increment: function() {
      count++;
      console.log(count);
    },
    decrement: function() {
      count--;
      console.log(count);
    }
  };
}
const counter = createCounter();
counter.increment(); // 1
counter.increment(); // 2

14. Error Handling

Robust applications need to handle unexpected issues gracefully. The try...catch statement allows you to "try" a block of code and "catch" any errors that occur, preventing your entire script from crashing.

try {
  // Code that might throw an error
  let result = riskyOperation();
  console.log(result);
} catch (error) {
  // This block runs if an error occurs
  console.error("An error occurred:", error.message);
} finally {
  // This block runs regardless of whether there was an error
  console.log("Execution finished.");
}

15. JSON

JSON (JavaScript Object Notation) is a lightweight format for storing and transporting data. It's often used when data is sent from a server to a web page. JavaScript provides built-in methods to work with JSON.

JSON.stringify()

Converts a JavaScript object into a JSON string.

const user = { name: "Alice", age: 30 };
const jsonString = JSON.stringify(user);
// Result: '{"name":"Alice","age":30}'

JSON.parse()

Converts a JSON string back into a JavaScript object.

const json = '{"name":"Bob","age":25}';
const userObject = JSON.parse(json);
console.log(userObject.name); // Bob

16. The Document Object Model (DOM)

The DOM is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects, allowing JavaScript to interact with the page.

Selecting Elements

// Get an element by its ID
const title = document.getElementById('main-title');

// Get the first element that matches a CSS selector
const firstItem = document.querySelector('.list-item');

// Get all elements that match a CSS selector
const allItems = document.querySelectorAll('.list-item');

Manipulating Elements

// Change the text content
title.textContent = 'New Title';

// Change the style
firstItem.style.color = 'blue';

// Add a CSS class
firstItem.classList.add('highlight');

// Create a new element
const newElement = document.createElement('div');
newElement.textContent = 'I am new!';
document.body.appendChild(newElement);

17. Events

Events are actions or occurrences that happen in the browser, such as a user clicking a button or a page finishing loading. You can use JavaScript to "listen" for these events and run code in response.

Event Listeners

The addEventListener() method is the modern, recommended way to handle events.

const myButton = document.getElementById('myButton');

function handleClick() {
  console.log('Button was clicked!');
}

// Attach the event listener
myButton.addEventListener('click', handleClick);

Common Events

  • click: The user clicks an element.
  • mouseover / mouseout: The mouse pointer moves over/out of an element.
  • keydown / keyup: The user presses/releases a key.
  • submit: A form is submitted.
  • DOMContentLoaded: The HTML document has been completely loaded and parsed.
